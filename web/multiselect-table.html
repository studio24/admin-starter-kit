About this example
  • This web component relies on the table and it's parent div (which makes the table scrollable on smaller screens) being wrapped in a custom element: <amplify-table-multiselect>. This adds the JavaScript which:
    • inserts the main checkbox for controlling all individual checkboxes in the table rows
    • adds a label to the main checkbox
    • handles the state of the main checkbox and the checkboxes in the table, including an indeterminate state if only some of the tble checkboxes are checked
  • The custom element is contained in a form, because it is assumed that selecting items would be followed by some sort of bulk action (e.g. deletion).
  • The main checkbox is placed before the table, which is a best practice recommendation for accessibility.
  • The default wording for the main checkbox label is "Select all". This can be overridden using the data-multiselect-label attribute on the custom element to make it more relevant to the table contents, as in this example.
