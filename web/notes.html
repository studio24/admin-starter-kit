About these examples
  • The role on the outermost container creates an ARIA live region when set to status or alert.
  • Only error messages should have a role of alert. The alert role should read out content that has changed, or bring the user's attention to it immediately, so it should not be used for static content or used regularly. Alerts, by definition, are disruptive. Lots of alerts at once or unnecessary alerts will create a bad user experience.
  • If the content of the note is advisory information not important enough to justify an alert, set the role to status.
