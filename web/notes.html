Notes examples

About these examples
  • Notes that appear in response to user actions need to be injected into one of the two empty containers that hold them: div id="site-aria-status" for status messages and div id="site-aria-alert" for alert messages.
  • The role on these containers creates an ARIA live region when set to status or alert.
  • Having them in the initial page markup is the most robust method for making sure screen reader users are aware when new content is added to the page. Because these ARIA live regions are always on the page and empty until needed, screen readers are primed to monitor them.
  • Only error messages should be added to div id="site-aria-alert", which has a role of alert. The user's attention should be brought to it immediately. Alerts, by definition, are disruptive; lots of alerts at once or unnecessary alerts will create a bad user experience.
  • If the content of the note is advisory information not important enough to justify an alert, add it to div id="site-aria-status", which has a role of status.
  • Don't forget to remove these notes from your production code.
